## Как я подхожу к проектам

```text
Business goal -> architecture -> working prototype -> validation -> proof -> launch
```

Для меня важно, чтобы AI-проект был не набором промптов, а воспроизводимой
системой:

- есть понятная задача и границы;
- есть архитектура и документация;
- есть рабочий код или проверяемый showcase;
- есть тесты, smoke-checks или аудит готовности;
- есть evidence trail: отчёты, кейсы, changelog, proof snapshots.
